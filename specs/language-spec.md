# Hana（花）— 日本語学習用プログラミング言語 仕様書 v0.2

## 1. 概要

### 1.1 目的
小学校中〜高学年を対象とした、スマホでリアルタイムに実行できる学習用プログラミング言語。

### 1.2 設計原則
- **日本語完結**: プログラム全体を日本語で記述可能。英数字の入力を要求しない。
- **記号ベースの構文**: 助詞を使わず、`=`と括弧で構造を表現。Python日本語版のような直感的な文法。
- **曖昧入力許容**: 全角/半角・表記揺れをトークナイザで自動正規化。
- **即時実行**: REPLおよび1行実行が可能。
- **小さな言語**: 予約語は20程度に収める。

### 1.3 ターゲット環境
- iOS / Android（React Native + Rust/WASM）
- Webブラウザ（WASM）
- ターミナル（ネイティブバイナリ）

## 2. 字句規則

### 2.1 文字エンコーディング
UTF-8。ソースコードは日本語文字を含む。

### 2.2 自動正規化（トークナイザ処理）
入力時に以下の変換を自動で行う。ユーザーは意識不要。

| 変換元 | 変換先 | 説明 |
|--------|--------|------|
| `０`〜`９` | `0`〜`9` | 全角数字 → 半角 |
| `＋` `－` `＊` `／` | `+` `-` `*` `/` | 全角演算子 → 半角 |
| `＝` | `=` | 全角イコール |
| `！` `？` | `!` `?` | 全角記号 |
| `（` `）` | `(` `)` | 全角括弧 → 半角 |
| `「` `」` | `"..."` | 鍵括弧 → 文字列リテラル |
| `［` `］` | `[...]` | 亀甲括弧 → 配列リテラル |
| `、` `。` | カンマ/ドット相当 | 日本語句読点を処理（文脈により） |
| `　` | ` ` | 全角スペース → 半角 |

### 2.3 日本語句読点の処理

`、`と`.`は同等に扱う。リスト区切りや文の区切りとして使用可能。

```
果物 = ［「りんご」。「バナナ」。「みかん」］    # 。をカンマとして扱う
果物 = ［「りんご」，「バナナ」，「みかん」］    # 、でもOK
```

ただし、文字列リテラル内の `。` は通常の文字として扱われる。

### 2.4 字句要素

#### 2.4.1 予約語

```
もし           if
そうでなければ    elif
違えば         else
繰り返す       for / while
回            繰り返し用
の間          while用
回数          繰り返し内暗黙変数
〜とは         def (関数定義)
戻す          return
表示          print
聞く          input
そして         and
または         or
ではない        not
真           true
偽           false
なし          null
```

#### 2.4.2 演算子

```
+  -  *  /        四則演算
%               剰余
=               代入
== != < > <= >=   比較
```

#### 2.4.3 区切り子

```
:    コロン（ブロック開始）。全角：も可。
()   括弧（関数呼び出し、式のグループ化）。全角（）も可。
[]   配列リテラル。全角［］も可。
改行  文末（セミコロン不要）
空白  トークン区切り
```

#### 2.4.4 リテラル

```
数値:   42, 3.14, -7
文字列: 「こんにちは」または "こんにちは"
真偽:   真, 偽
配列:   ［1, 2, 3］または [1, 2, 3]
なし:   なし
```

#### 2.4.5 識別子
- 日本語、英数字、アンダースコア `_` を使用可能
- 数字開始不可
- 例: `名前`, `合計金額`, `counter`, `フルーツリスト`

### 2.5 コメント

```
# これは一行コメント
```

## 3. 構文規則

### 3.1 ブロック構造
インデント（空白またはタブ）でブロックを表現。Pythonと同じルール。

```
もし 年齢 >= 12 なら：
　表示「中学生以上です」
```

- `:`（全角`：`も可）でブロック開始を宣言
- 直後の改行以降、インデントが深い行がブロック本体
- インデントが戻った時点でブロック終了
- インデント量は混在不可（ファイル内で統一）

### 3.2 文
1行1文を基本とする。複数文を書く場合は `;` で区切る。

```
名前 = 「タロウ」; 年齢 = 12
```

## 4. 文法（BNF風）

```
プログラム    = 文*
文          = 代入文 | 関数呼び出し文 | 条件文 | 繰り返し文 | 関数定義文 | 式文 | コメント

# 代入
代入文       = 識別子 "=" 式

# 出力（関数呼び出し構文）
表示文       = "表示" "(" 式 ("," 式)* ")"
            | "表示" 式

# 入力
入力文       = 識別子 "=" "聞く" "(" 式 ")"

# 条件
条件文       = "もし" 式 "なら" ":" 改行 ブロック
              ("そうでなければ" 式 "なら" ":" 改行 ブロック)*
              ("違えば" ":" 改行 ブロック)?

# 繰り返し（回数）
繰り返し文   = 式 "回" "繰り返す" ":" 改行 ブロック

# 繰り返し（条件）
繰り返し文   = 式 "の間" "繰り返す" ":" 改行 ブロック

# 関数定義
関数定義文    = 識別子 "とは" "(" 識別子 ("," 識別子)* ")" ":" 改行 ブロック

# 式
式          = 代入式 | 論理式
論理式       = 比較式 ("または" | "そして" 比較式)*
比較式       = 加減式 ("==" | "!=" | "<" | ">" | "<=" | ">=" 加減式)*
加減式       = 乗除式 ("+" | "-" 乗除式)*
乗除式       = 単項式 ("*" | "/" | "%" 単項式)*
単項式       = "-" 単項式 | "ではない" 単項式 | 後置式
後置式       = 基本式 ("の" 識別子 | "の" 式 "番目")*
基本式       = 数値リテラル | 文字列リテラル | 真偽リテラル | 配列リテラル | 識別子
            | "真" | "偽" | "なし"
            | 関数呼び出し
            | "(" 式 ")"

# 関数呼び出し
関数呼び出し  = 識別子 "(" 引数リスト? ")"
引数リスト    = 式 ("," 式)*

# 配列
配列リテラル  = "[" 式 ("," | "。" | "、") 式* "]"
```

## 5. 標準組み込み関数

### 5.1 入出力

| 構文 | 意味 | 例 |
|------|------|-----|
| `表示(X)` | Xを出力して改行 | `表示「hello」` |
| `表示(X, Y, ...)` | 複数値をスペース区切りで出力 | `表示「答えは」、42` |
| `聞く(メッセージ)` | 入力ダイアログを表示 | `名前 = 聞く「名前は？」` |

### 5.2 文字列

| 関数 | 意味 | 例 |
|------|------|-----|
| `文字数(X)` | 文字列長 | `文字数「いろは」` |
| `繋ぐ(X, Y, ...)` | 結合 | `繋ぐ「A」「B」` |
| `切り取る(X, 開始, 終了)` | 部分文字列 | `切り取る「abcdef」、2、4` |

### 5.3 数学

| 関数 | 意味 | 例 |
|------|------|-----|
| `絶対値(X)` | 絶対値 | `絶対値(-5)` |
| `四捨五入(X)` | 小数丸め | `四捨五入(3.7)` |
| `乱数(最小, 最大)` | 乱数生成 | `乱数(1, 6)` |
| `余り(X, Y)` | 剰余 | `余り(10, 3)` |
| `切り上げ(X)` | 切り上げ | `切り上げ(3.1)` |
| `切り下げ(X)` | 切り下げ | `切り下げ(3.9)` |

### 5.4 リスト

| 関数 | 意味 | 例 |
|------|------|-----|
| `[A, B, C]` | 配列作成 | `果物 = ［「りんご」。「バナナ」］` |
| `要素数(X)` | 長さ | `要素数(果物)` |
| `追加(リスト, 値)` | 末尾追加 | `追加(果物。「みかん」)` |
| `削除(リスト, 番号)` | 要素削除 | `削除(果物、0)` |

### 5.5 型変換

| 関数 | 意味 | 例 |
|------|------|-----|
| `数にする(X)` | 数値変換 | `数にする「42」` |
| `文字にする(X)` | 文字列変換 | `文字にする(42)` |

## 6. 型システム

動的型付け。実行時に型が決定される。

| 型 | 説明 | 例 |
|----|------|-----|
| 数値 | 整数および小数 | `42`, `3.14` |
| 文字列 | テキスト | `「こんにちは」` |
| 真偽 | 真/偽 | `真`, `偽` |
| 配列 | 順序付きコレクション | `［1, 2, 3］` |
| 関数 | 第一級オブジェクト | `足すとは：(a)：戻す a` |
| なし | null値 | `なし` |

### 6.1 暗黙の型変換
```
数値 + 文字列 → 文字列   # 表示(「答えは」、42)  ← OK
文字列 - 数値 → エラー    # 「3」- 1    ← NG（数にする が必要）
```

## 8. 関数

### 8.1 定義
```
挨拶するとは(名前)：
　表示「{名前}さん、こんにちは！」
```

複数引数:
```
足すとは(a, b)：
　戻す a + b
```

### 8.2 呼び出し
```
挨拶する「タロウ」
結果 = 足すする(3, 5)
```

### 8.3 戻り値
`戻す` で明示的に返す。`戻す` がない場合は `なし` を返す。

### 8.4 プロパティアクセス（後置式）
`の` を使ってオブジェクト/文字列のプロパティや配列要素にアクセス。

```
名前 = 「いろは」
表示「{名前}の文字数は{名前の文字数}です」
表示 名前の0番目      # → い

果物 = ［「りんご」。「バナナ」］
表示 果物の0番目       # → りんご
表示 果物の要素数      # → 2
```

## 9. 文字列補間

`{}` で変数や式を埋め込み可能。

```
名前 = 「タロウ」
「{名前}は{12 + 3}歳です」と表示
# → タロウは15歳です
```

文字列補間は文字列リテラル内でのみ有効。`{}` 内には任意の式を書ける。

## 10. エラーメッセージ（子供向け）

エラーメッセージは日本語で、原因と修正方法を提示。

| エラー種別 | メッセージ例 |
|------------|-------------|
| 未定義変数 | `「なまえ」という変数はまだ作られていません。「名前」の入力ミスかもしれません。` |
| 型エラー | `「こんにちは」から2を引くことはできません。数値を使ってください。` |
| ゼロ除算 | `0で割ることはできません。` |
| 無限ループ検出 | `繰り返しが10000回を超えました。間違っていませんか？` |
| 引数エラー | `表示には1つ以上の引数が必要です。表示(「hello」) のように書いてみてください。` |

```

　表示「正解！」
　正解数 = 正解数 + 1

　表示「残念！答えは7です」

　表示「正解！」
```

## 13. v0.1 → v0.2 変更履歴

### 変更点: 助詞廃止・記号ベース構文への移行

**変更前 (v0.1):**
```
名前は「タロウ」
「{名前}さん」と表示
「名前は？」と聞く
　→ 名前に入れる
```

**変更後 (v0.2):**
```
名前 = 「タロウ」
表示「{名前}さん」
名前 = 聞く「名前は？」
```

**理由:**
- 助詞（は/が/を/と）の使い分けが子供にとって混乱の元
- 曖昧解析の複雑さを削減
- 既存のプログラミング言語（Python等）の知識が移行しやすい
- `=`と括弧という普遍的な記号で構造を表現
