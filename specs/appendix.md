## 表記揺れの正規化テーブル

トークナイザが以下の表記揺れを同一の予約語として扱う。

### 条件分岐
| 正式名 | 許容表記 |
|--------|---------|
| もし | もし, モシ |
| なら | なら, ナラ |
| そうでなければ | そうでなければ, またはもし, 又はもし |
| 違えば | 違えば, ちがえば |

### 繰り返し
| 正式名 | 許容表記 |
|--------|---------|
| 回 | 回, カイ |
| 繰り返す | 繰り返す, くりかえす, クリカエス |
| の間 | の間, のあいだ |
| 回数 | 回数（繰り返し内で暗黙定義される変数） |

### 関数
| 正式名 | 許容表記 |
|--------|---------|

## 演算子の優先順位（低→高）

```
1. または    (or)
2. そして    (and)
3. ではない   (not, 単項)
4. 比較演算   == != < > <= >=
5. + -       (加減)
6. * / %     (乗除余)
7. 単項 -     (負号)
8. の        (プロパティアクセス / 配列インデックス)
9. ()        (括弧内の式)
```

## 文字列補間の詳細

`{}` 内には任意の式を書ける。

```
「{3 + 4}の答えは{3 + 4 * 2}」と表示
# → 7の答えは11
```

ネスト不可（v0.1）。`{「abc」}` はエラー。
文字列補間内の式から文字列を返す場合:

```
名前は「タロウ」
「{名前}」と表示     # OK: 「タロウ」と表示
```

## 「回繰り返す」の暗黙変数

`N回繰り返す` のブロック内では `回数` が自動的に定義される（0始まり）。

```
3回 繰り返す：
　「{回数 + 1}回目」と表示
# 出力:
# 1回目
# 2回目
# 3回目
```

`回数` は読み取り専用。代入しようとするとエラー。

## 「聞く」の動作

```
「名前は？」と聞く
　→ 名前に入れる
```

実行時の動作:
1. 入力ダイアログ（モバイル）またはプロンプト（ターミナル）を表示
2. ユーザーが入力して確定
3. 入力値を文字列として `→` の左の識別子に代入
4. 数値として扱いたい場合は `Xを数にする` で変換

```
「年齢は？」と聞く
　→ 入力年齢に入れる
年齢は 入力年齢を数にする
```
